# Hyper-V-Network-Lab
I built a virtual network in Hyper‑V using Windows, Kali Linux, and a pfSense firewall to simulate a small enterprise environment. Each system was assigned a static IP and connected through an internal switch, allowing me to test communication, understand traffic flow, and practice real networking fundamentals.
Objective- To design a small, isolated virtual network that demonstrates core networking concepts using multiple operating systems and a firewall. The goal was to configure static IPs, validate host‑to‑host communication, and understand how traffic flows inside a protected LAN environment.
Tools used : Hyper‑V – Virtualization platform for building the network
Windows 10 VM – Workstation for IP configuration and connectivity tests
Kali Linux VM – Linux host for networking commands and verification
pfSense Firewall – Open‑source firewall acting as the network backbone
Hyper‑V Virtual Switch Manager – Created the internal LAN
Windows CMD / PowerShell – Used for ipconfig and ping tests
Linux Terminal – Used for ip a and connectivity checks
What I learned
How to build and manage multiple VMs in Hyper‑V
How internal virtual switches create isolated LANs
How to assign and verify static IP addresses
How Windows and Linux handle networking differently
How to test communication using ping, ipconfig, and ip a
How a firewall fits into a virtual network
How traffic flows between hosts on the same subnet
Foundational skills for SOC, blue team, and IT networking
